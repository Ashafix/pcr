FAQ

    Who are the people behind the program?

    This program was developed as a collaboration between David Zeevi and Maximilian Peters. David Zeevi had the original idea and is responsible for all the wet lab implementation, Maximilian Peters is responsible for all the code and software architecture behind it. You can contact us by email.

    How does it work?

    A lot of Python code identifies the sequence for which primers should be designed. Primer3 creates an initial list of possible primers which are checked against the human hg19 genome by isPCR. Depending on the settings either different primer pairs or hemi-nested primers are suggested. Since this program was developed independent of any research institutes and we do not have access to a computing center, we run the programs on Amazon Web Service. The main compute server is only started when needed to save money.

    What are all those settings?

    The standard settings are the ones which were validated and optimized in wet-lab experiments. You can create your own primer3 input file, upload it and run the program with those settings, for example if you do not care about amplicon length, you can set PRIMER_PAIR_WT_PRODUCT_SIZE_LT and PRIMER_PAIR_WT_PRODUCT_SIZE_GT to zero which dramatically reduces run time but results in different primer pairs. For a general explanation of all the primer3 settings, please have a look at their manual.

    The other parameters used by Hemi-NeSTR are:
    Maximum length of repeats which are searched: Specifies the maximum length of the repeat unit, e.g. 3 will search for repeats of any trinucleotide
    Maximum number of primers to be returned: The maximum number of primer pairs which are returned
    Maximum similarity of primer pairs: The maximum similarity of the return primer pairs, 1 equals up to 100% identity, 0.5 would be up to 50% identity, and 0 would be 0% identity.
    Search for nested primers: always: default setting, will alsways try to find hemi-nested primers, if needed: only if the first run didn't return enough primer pairs, never: even if the first run didn't return enough primer pairs, no 2nd attempt will be undertaken.

    Which tools are used by the program?

    Primers are designed with Primer3 and the specificity is checked with isPCR.

    How fast is it?/How much time does it take?

    It takes approximately 2-3 minutes per sequence to finish. All your jobs are handled in parallel by the server. Most of the time is consumed by primer3, for nested primer3 is has to run at least twice. Each primer3 run takes between 1-3 minutes. Depending on the number of sequences either a 2 or 8 core server is started in order to find an acceptable ratio between run time and costs. As a rule of thumb: 2 sequence: 2-3 min, 8: sequences: 8-12 min, 16 sequences: 4-6 min, 32 sequences: 8-12 min.

    How reliable are the results?

    Over the course of one year, Hemi-NeSTR was used to design an average of 16.6 hemi-nesting STR-flanking amplicons in each of 39 different 4Mb sized genomic regions. For each region, 78.4%+/-18.7% of the designed primer pairs successfully amplified their respective STR targets. After linkage analysis, an average of 46.8%+/-21.4% of the designed STR amplicons were identified as informative for single cell haplotype analysis. Thus far, 17 novel mutation-flanking STR panels were applied in live PGD cases. Generally, each panel consisted of 5-6 informative STR markers for haplotype analysis of single cell embryo biopsies. Using hemi-nested PCR amplification strategies, 97.2%+/-6.1% of the markers in each panel were successfully amplified with Hemi-NeSTR designed PCR primers. In all PGD cases, a diagnosis was achieved for each biopsy and currently there are 8 ongoing pregnancies from Hemi-NeSTR facilitated PGD.

    What are the hardware requirements?

    Any Linux machine with 4 GB of RAM can handle the program. For Windows computers a virtual machine is needed. If you need help setting it up, let use know.

    Can I use this program for other genomes?

    I guess so, we never tested it with other genomes but all tools can handle other genomes as well.
